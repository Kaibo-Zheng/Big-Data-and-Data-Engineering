# 《大数据与数据工程》课程大作业要求

本文件整理《大数据与数据工程》课程大作业要求，用于报告撰写、材料打包和提交检查。

## 作业主题

学生需结合本学期课程学习内容和课程实验，围绕个人专业方向或兴趣完成一个课程大作业，并提交一篇课程大作业报告。

## 选题与实现要求

- 作业一人一题。
- 作业内容可包括 MySQL 数据库应用、数据平台构建、数据获取、数据存储、数据处理、数据分析、数据可视化等。
- 可以参考课程实验形式，对自选案例进行修改和实现；也可以通过网络查找课程相关应用案例，并完成案例复现。
- 编程语言应限于课程相关语言。
- 源程序应命名规范、代码规范，并包含清晰的功能注释。

可参考以下资源查找案例或数据：

## 报告要求

- 报告篇幅至少 4 页。
- 报告建议图文并茂，排版规范。
- 报告内容应能说明案例背景、数据来源、实现过程、实验结果和改进思路。

报告文本格式要求如下：

- 报告标题：黑体，小四字体。
- 姓名、学号、专业：宋体，五号字体。
- 摘要：宋体，小五字体。
- 关键词：3 到 5 个，宋体，小五字体。
- 正文：段落首行缩进两个汉字，字体为宋体，五号字体。

## 报告结构

报告正文建议包含以下部分：

1. 案例背景介绍
2. 案例相关数据集描述
3. 案例涉及的模型与方法实现
4. 案例实现过程
5. 改进思路
6. 结论
7. 参考文献
8. 附录：网盘链接地址

如案例不涉及数据集或模型方法，可根据实际情况调整相应章节。

## 参考文献格式

出版物类文献格式：

```text
[1] 作者，文献标题，会议或期刊或出版社名称，年月日。
```

网页类文献格式：

```text
[2] 作者或昵称，文献标题，网络地址，年月日。
```

## 提交材料

报告相关的 PPT、参考文献、源程序、测试结果和数据集需要压缩为：

```text
学号+姓名.zip
```

压缩包需上传至网盘，建议使用百度网盘或同济云盘，有效期至少一个月，并将网盘链接放入报告附录。

报告文件命名为：

```text
学号+姓名.doc/pdf
```

报告文件需压缩为：

```text
学号+姓名-报告.zip
```

最终在作业截止日前上传至 `canvas.tongji.edu.cn`。
